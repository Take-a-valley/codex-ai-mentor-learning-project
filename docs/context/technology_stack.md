このファイルには、プロジェクトで使用する技術スタックを記録します。

| 領域 | 技術 | 採用理由 | 未決事項 |
|---|---|---|---|
| エディタ | 未定 | 未定 | 未定 |
| AIツール | 未定 | 未定 | 未定 |
| フロントエンド | Next.js | クライアント希望。画面中心のWebアプリを構築しやすい | バージョン、UIライブラリ、状態管理 |
| バックエンド | Spring Boot | クライアント希望。業務アプリ向けのAPI実装を学びやすい | バージョン、API方式、認証実装方式 |

| 認証・認可 | メールログイン、パスワードリセット、ロール別権限 | クライアント希望。管理者、メンバー、閲覧者の3ロールで管理する | メール認証、認証実装方式、詳細な認可方式 |

## 技術選定時に確認すること

- なぜその技術を使うのか
- 学習目的に合っているか
- 実務での採用例があるか
- 初学者にとって難しすぎないか
- 長期運用・保守に向いているか
- 代替技術と比べたメリット・デメリットは何か
- プロジェクトの要件に合っているか
