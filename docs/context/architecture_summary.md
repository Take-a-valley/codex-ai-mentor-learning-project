# Architecture Summary

## 現在の状態

- タスク管理機能の設計書は `docs/architecture/Task management function/` 配下で管理する
- 基本設計書は `docs/architecture/Task management function/Basic design/` に集約済み
- 詳細設計書は今後 `docs/architecture/Task management function/Detail design/` に作成する
- 要件定義書、基本設計書、DB設計書、API設計書、画面設計書、認証・認可設計書、フロントエンド設計書、バックエンド設計書、テスト方針書は作成済み
- 基本設計書一式の横断レビューでは大きな問題なし
- 残修正候補は、画面設計書のプロジェクト削除配置、画面/API対応表、API設計書の認可表タイポ
- 設計書は `/docs/architecture/` 配下にユーザーが作成する
- AIは `/docs/architecture/` 配下のファイルを直接編集しない
- AIはレビュー、助言、雛形案の提示のみ行う

## 想定アーキテクチャ

- フロントエンド: Next.js
- バックエンド: Spring Boot
- データストア: MySQL
- 認証・認可: メールログイン、パスワードリセット、システム管理者・プロジェクト管理者・メンバー・閲覧者
- 添付ファイル: 初期MVPではDB保存
- 初期対象機能: 複数プロジェクト対応のカンバン方式タスク管理

## 設計方針

- 初期MVPではカンバン列を `ToDo / In Progress / Review / Done` の固定列とする
- ステータス列のカスタマイズは将来拡張とする
- システム全体ロールとプロジェクト内ロールを分ける
- プロジェクト作成はシステム管理者のみ行える
- プロジェクト作成直後は、作成したシステム管理者を対象プロジェクトのプロジェクト管理者として登録する
- システム管理者と対象プロジェクトのプロジェクト管理者のみプロジェクト管理とタスク削除を行える
- メンバーはタスク作成・編集・移動を行える
- 閲覧者は閲覧のみ行える
- 閲覧者はコメントを閲覧できる
- パスワードリセットは、メールにリセット用URLを送る方式とする
- パスワードリセットURLの有効期限は30分とする
- コメント削除は管理者のみ行える
- コメント編集は投稿者本人のみ行える
- 添付ファイルは1ファイルあたり5MBまでとする
- 添付可能なファイル種別は、画像、PDF、テキスト、Office系ファイルとする
- プロジェクトメンバーはシステム管理者またはプロジェクト管理者がメール招待する
- メール招待URLの有効期限は24時間とする
- 未登録ユーザーは招待URLからユーザー登録後、プロジェクトに参加できる
- システム管理者またはプロジェクト管理者はメンバーのロールを変更できる
- 招待時の初期ロールはメンバー固定とする
- システム管理者自身のロール変更は、他のシステム管理者が存在する場合のみ許可する
- プロジェクト管理者自身のロール変更は、同一プロジェクト内に他のプロジェクト管理者が存在する場合のみ許可する
- システム管理者またはプロジェクト管理者が0人になる操作は禁止する
- 主要データの削除は論理削除とする
- API認証方式はセッション方式とする
- APIレスポンスはJSON形式を基本とする
- 添付ファイル取得APIのみファイルデータを返す
- タスク作成・編集はモーダルで行う
- カンバン画面のドラッグアンドドロップによるStatus変更を初期MVPに含める

## 決定事項

- AIは実装代行ではなくメンターとして使用する
- 設計書は `/docs/architecture/` にユーザーが作成する
- AIは `/docs/context/` と `/docs/logs/` のMarkdownのみ直接編集できる
- 初期MVPは複数プロジェクト対応のカンバン型タスク管理
- 使用技術候補は Next.js、Spring Boot、MySQL
- 認証はメールログインとパスワードリセット
- 権限はシステム管理者、プロジェクト管理者、メンバー、閲覧者で整理する
- パスワードリセットURLの有効期限は30分
- 添付ファイルはDB保存、1ファイルあたり5MBまで
- プロジェクトメンバーはシステム管理者またはプロジェクト管理者がメール招待する
- メール招待URLの有効期限は24時間
- 未登録ユーザーは招待URLからユーザー登録後、プロジェクトに参加できる
- システム管理者またはプロジェクト管理者はメンバーのロールを変更できる
- 招待時の初期ロールはメンバー固定
- 管理者が0人になる操作は禁止
- 主要データの削除は論理削除とする
- API認証方式はセッション方式とする
