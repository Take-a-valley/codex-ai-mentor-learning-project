## 想定アーキテクチャ

- フロントエンド: Next.js
- バックエンド: Spring Boot
- データストア: MySQL
- 認証・認可: メールログイン、パスワードリセット、システム管理者・プロジェクト管理者・メンバー・閲覧者
- 添付ファイル: 初期MVPではDB保存
- 初期対象機能: 複数プロジェクト対応のカンバン方式タスク管理

## 設計方針

- 初期MVPではカンバン列を `ToDo / In Progress / Review / Done` の固定列とする
- ステータス列のカスタマイズは将来拡張とする
- システム全体ロールとプロジェクト内ロールを分ける
- プロジェクト作成はシステム管理者のみ行える
- プロジェクト作成直後は、作成したシステム管理者を対象プロジェクトのプロジェクト管理者として登録する
- システム管理者と対象プロジェクトのプロジェクト管理者のみプロジェクト管理とタスク削除を行える
- メンバーはタスク作成・編集・移動を行える
- 閲覧者は閲覧のみ行える
- 閲覧者はコメントを閲覧できる
- パスワードリセットは、メールにリセット用URLを送る方式とする
- パスワードリセットURLの有効期限は30分とする
- コメント削除は管理者のみ行える
- コメント編集は投稿者本人のみ行える
- 添付ファイルは1ファイルあたり5MBまでとする
- 添付可能なファイル種別は、画像、PDF、テキスト、Office系ファイルとする
- プロジェクトメンバーはシステム管理者またはプロジェクト管理者がメール招待する
- メール招待URLの有効期限は24時間とする
- 未登録ユーザーは招待URLからユーザー登録後、プロジェクトに参加できる
- システム管理者またはプロジェクト管理者はメンバーのロールを変更できる
- 招待時の初期ロールはメンバー固定とする
- システム管理者自身のロール変更は、他のシステム管理者が存在する場合のみ許可する
- プロジェクト管理者自身のロール変更は、同一プロジェクト内に他のプロジェクト管理者が存在する場合のみ許可する
- システム管理者またはプロジェクト管理者が0人になる操作は禁止する
- 主要データの削除は論理削除とする
- API認証方式はセッション方式とする
- DBはMySQL 8系を別途採用する
- DB以外の開発環境は、現在PCにインストールされているものを極力採用する
- Node.jsは `24.14.1`、npmは `11.11.0`、Javaは `25`、Mavenは `3.9.9`、Gitは `2.48.1.windows.1` を採用候補とする
- Spring Bootは `4.1.x` 系を採用候補とする
- Next.jsは `16.x` 系または作成時点の `latest` を採用候補とする
- DBアクセス方式はSpring Data JPAを採用する
- DBマイグレーションはFlywayを採用する
- 開発環境ではDocker Composeを採用する
- 開発用メール確認はMailpitを採用する
- セッション有効期限は最終アクセスから120分とする
- CSRF対策はSpring Securityで検証し、CSRFトークン取得APIを用意する
- UIはTailwind CSSを基本とし、必要に応じてshadcn/uiを採用する
- サーバー状態はTanStack Queryで管理し、画面状態はReact標準の状態管理で扱う
- フロントエンドテストはVitest + React Testing Libraryを採用候補とする
- E2EテストはPlaywrightを採用候補とする
- APIレスポンスはJSON形式を基本とする
- 添付ファイル取得APIのみファイルデータを返す
- タスク作成・編集はモーダルで行う
- カンバン画面のドラッグアンドドロップによるStatus変更を初期MVPに含める
- ワイヤーフレームはHTML形式で作成し、主要画面を1ページで確認できる構成とする
- Next.jsはApp Routerを採用する方針とする
- shadcn/uiは必要箇所のみ使用する方針とする
- パスワードルールは8文字以上72文字以下、英字と数字を各1文字以上、記号任意を推奨方針とする
- ログイン失敗回数制限は初期MVP対象外とし、将来拡張とする方針とする
- 招待・パスワードリセットの期限切れ処理は参照時に判定する方針とする
- JavaのID型はLong、日時はLocalDateTime、期限日はLocalDateを基本方針とする
- tokenは推測困難なランダム文字列としてDB保存する方針とする
- ControllerではSpring Securityの認証情報からログインユーザーを取得し、ServiceへログインユーザーIDまたは情報を渡す方針とする
