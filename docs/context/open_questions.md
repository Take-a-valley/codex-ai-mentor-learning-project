## 解決済み事項

- 作成するシステムは開発支援アプリケーション
- 想定利用者は小規模開発チーム
- 初期対象はカンバン方式のタスク管理
- 複数プロジェクト作成・切り替えを初期MVPに含める
- タスク管理には担当者、期限、優先度、コメント、添付ファイルを含める
- 認証後の権限は管理者、メンバー、閲覧者に分ける
- 使用技術候補は Next.js、Spring Boot、MySQL、メールログイン
- タスクのステータス列は初期固定、将来拡張で変更可能にする
- 管理者のみプロジェクト管理可、メンバーはタスク操作可、閲覧者は閲覧のみ
- 添付ファイルはアップロード、ダウンロード、削除を初期MVPに含める
- パスワードリセットを初期MVPに含める
- 初期固定ステータス列は ToDo / In Progress / Review / Done
- メンバーはタスクの作成・編集・移動ができるが、削除は管理者のみ
- 添付ファイルは初期MVPではローカル保存
- パスワードリセットはメールにリセット用URLを送る方式
- コメントはメンバーが作成・編集でき、削除は管理者のみ
- 添付ファイルは1ファイルあたり5MBまで
- 添付可能なファイル種別は画像、PDF、テキスト、Office系ファイル
- プロジェクトメンバーは管理者がメール招待する
- パスワードリセットURLの有効期限は30分
- コメント編集は投稿者本人のみ可能
- 招待URLの有効期限は24時間
- 未登録ユーザーは招待URLからユーザー登録後、プロジェクトに参加できる
- 管理者はメンバーのロールを変更できる
- 閲覧者はコメントを閲覧できる
- 招待時の初期ロールはメンバー固定
- 管理者は他に管理者がいる場合のみ自分自身のロールを変更できる
- プロジェクト内の管理者が0人になる操作は禁止する
- 要件定義書は画面・API・DB設計にそのまま進める詳しさで作成する
- 要件定義書はシステム概要から順番に作成する
- AIレビューは要件定義書を一通り書いてから行う
- 要件定義書ファイルは章構成を確認してからユーザーが作成する
- AIは要件定義書の見出し案と各章に書く内容を提示する
