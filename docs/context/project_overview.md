# Project Overview

## プロジェクト名

未定

## 目的

システム開発を行う上で必要最低限の機能を集約した、開発支援アプリケーションを作成する。

## 作成するシステム

小規模開発チーム向けの開発支援アプリケーション。

初期リリースでは、複数プロジェクトを作成・切り替えできるカンバン方式のタスク管理機能を作成する。

将来的には以下の機能を段階的に追加する。

- Wikiによるドキュメント管理
- WBS、ガントチャート
- プロジェクト内チャットの統合管理

## 背景

開発プロジェクトでは、タスク管理、ドキュメント管理、進捗管理、コミュニケーションが複数ツールに分散しやすい。

本プロジェクトでは、これらを段階的に統合できる開発支援アプリケーションを題材にして、要件定義から設計、実装、レビュー、保守までを学ぶ。

## 学習目的

例：

- 要件定義の流れを理解する
- 設計書の種類と役割を理解する
- 選定した技術スタックを使ったWebシステム構成を理解する
- AIに丸投げせず、自分で考えて実装する力を身につける
- レビュー観点とデバッグ力を身につける

## 想定利用者

小規模開発チーム。

認証後の権限は以下の3種類を想定する。

- 管理者
- メンバー
- 閲覧者
