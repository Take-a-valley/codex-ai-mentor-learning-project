# Scope

## 初期スコープ方針

本プロジェクトでは、最初から全機能を完成させるのではなく、段階的に実装します。

まずは学習目的と実現可能性を踏まえ、MVPを定義してください。

## MVP候補

複数プロジェクト対応のカンバン方式タスク管理。

初期MVPには以下を含める。

- 複数プロジェクトの作成・切り替え
- タスク作成・編集・削除
- タスクのステータス移動
- 固定ステータス列: ToDo / In Progress / Review / Done
- 担当者
- 期限
- 優先度
- コメント
- コメント編集は投稿者本人のみ可能
- コメント削除は管理者のみ可能
- 添付ファイルのアップロード、ダウンロード、削除
- 添付ファイルの容量制限: 1ファイルあたり5MB
- 添付可能なファイル種別: 画像、PDF、テキスト、Office系ファイル
- 管理者、メンバー、閲覧者の権限区分
- 閲覧者はコメントを閲覧可能
- パスワードリセット
- パスワードリセット用URLのメール送信
- パスワードリセットURLの有効期限: 30分
- 管理者によるプロジェクトメンバーのメール招待
- メール招待URLの有効期限: 24時間
- 招待時の初期ロール: メンバー
- 未登録ユーザーの招待URL経由登録
- 管理者によるメンバーのロール変更
- 管理者自身のロール変更は他の管理者が存在する場合のみ可能
- 管理者が0人になる操作は禁止

## 後続候補

- Wikiによるドキュメント管理
- WBS、ガントチャート
- プロジェクト内チャットの統合管理

## 対象外候補

未定。
