# 画面設計書

## 1. 本書の目的

本書は、タスク管理機能の要件定義書、基本設計書、DB設計書、API設計書をもとに、初期MVPで必要となる画面構成、画面遷移、表示項目、操作、利用API、ロール別表示制御を整理することを目的とする。

後続工程であるフロントエンド詳細設計、実装、テスト設計へ進むために、各画面の役割と操作範囲を明確にする。

## 2. 対象範囲

本書では、初期MVPであるカンバン方式タスク管理機能に必要な画面を対象とする。

対象画面は以下とする。

- ログイン画面
- パスワードリセット申請画面
- パスワード再設定画面
- プロジェクト一覧画面
- プロジェクト作成画面
- プロジェクト編集画面
- カンバン画面
- タスク詳細画面
- メンバー管理画面
- メンバー招待画面
- 招待参加画面

以下は初期MVPでは対象外とする。

- Wiki画面
- WBS、ガントチャート画面
- プロジェクト内チャット画面
- 通知画面
- ステータス列カスタマイズ画面
- 外部ストレージ連携画面

## 3. 画面設計方針

画面は、ログイン状態、システムロール、プロジェクト内ロールに応じて表示内容と操作可否を制御する。

ただし、画面側の制御はユーザー体験向上のための補助であり、最終的な認可チェックはバックエンドAPI側で必ず行う。

画面上では、権限のない操作は原則として非表示または無効化する。

削除操作やロール変更など、影響が大きい操作は確認ダイアログを表示する。

API通信中はローディング状態を表示する。

データが存在しない場合は空状態を表示する。

APIエラー発生時は、ユーザーが次に取るべき行動を判断できるエラーメッセージを表示する。

## 4. 共通画面構成

認証後の画面では、以下の共通レイアウトを使用する。

| 領域 | 内容 |
|---|---|
| ヘッダー | アプリ名、現在のプロジェクト名、ログインユーザー情報、ログアウト |
| サイドバー | プロジェクト一覧、タスク管理への導線 |
| メイン領域 | 各画面の主要コンテンツ |
| モーダル | 作成、編集、削除確認、招待などの補助操作 |

初期MVPでは、Wiki、WBS、チャットへのメニューは表示しない。

将来拡張を見据える場合でも、未実装機能への導線は初期MVPでは非表示とする。

## 8. SCR-001 ログイン画面

### 画面目的

ユーザーがメールアドレスとパスワードでログインする。

### 表示項目

| 項目 | 内容 |
|---|---|
| メールアドレス入力 | ログイン用メールアドレス |
| パスワード入力 | ログイン用パスワード |
| ログインボタン | 入力内容でログインする |
| パスワードリセットリンク | パスワードリセット申請画面へ遷移する |

### 操作

| 操作 | 内容 | 利用API |
|---|---|---|
| ログイン | メールアドレスとパスワードで認証する | POST /api/auth/login |
| パスワードリセットへ遷移 | パスワードリセット申請画面へ移動する | なし |

### 入力チェック

| 項目 | チェック内容 |
|---|---|
| メールアドレス | 必須、メール形式 |
| パスワード | 必須 |

### エラー表示

| 状態 | 表示内容 |
|---|---|
| メール形式不一致 | メールアドレスにメール形式と異なる値が入力されている旨のメッセージを表示する |
| 認証失敗 | メールアドレスまたはパスワードが正しくない旨を表示する |
| メールアドレス未入力 | メールアドレスが入力されていない旨を表示する |
| パスワード未入力 | パスワードが入力されていない旨を表示する |

| 操作 | 内容 | 利用API |

| 項目 | チェック内容 |
|---|---|
| メールアドレス | 必須、メール形式 |

## 10. SCR-003 パスワード再設定画面

### 画面目的

パスワードリセットURLからアクセスしたユーザーが、新しいパスワードを設定する。

### 表示項目

| 項目 | 内容 |
|---|---|
| 新しいパスワード入力 | 再設定するパスワード |
| 新しいパスワード確認入力 | 入力確認用パスワード |
| 再設定ボタン | パスワードを更新する |

### 操作

| 操作 | 内容 | 利用API |
|---|---|---|
| パスワード再設定 | tokenと新しいパスワードで再設定する | POST /api/password-reset/confirm |

### 入力チェック

| 項目 | チェック内容 |
|---|---|
| token | URLから取得できること |
| 新しいパスワード | 必須 |
| 新しいパスワード確認 | 新しいパスワードと一致すること |

### エラー表示

| 状態 | 表示内容 |
|---|---|
| tokenなし | 無効なURLである旨を表示する |
| token期限切れ | リセットURLの有効期限が切れている旨を表示する |
| token使用済み | リセットURLが使用済みである旨を表示する |
| パスワード未入力 | 新しいパスワードが入力されていない旨を表示する |
| パスワード不一致 | 新しいパスワードは一致していない旨を表示する |

### 画面目的

| プロジェクト編集ボタン | プロジェクト編集画面へ遷移 |

## 14. SCR-007 カンバン画面

### 画面目的

対象プロジェクトのタスクをStatus別に表示し、タスクの作成、編集、ステータス変更、削除を行えるようにする。

### 表示項目

| 項目 | 内容 |
|---|---|
| プロジェクト名 | 現在表示中のプロジェクト |
| Status列 | ToDo / In Progress / Review / Done |
| タスクカード | Summary、Assignee、Priority、Due date |
| タスク作成ボタン | 新規タスク作成 |
| メンバー管理リンク | メンバー管理画面へ遷移 |
| プロジェクト編集リンク | プロジェクト編集画面へ遷移 |

### 操作

| 操作 | 内容 | 利用API |
|---|---|---|
| タスク一覧取得 | Status別のタスク一覧を取得する | GET /api/projects/{projectId}/tasks |
| タスク作成 | 新規タスクを作成する | POST /api/projects/{projectId}/tasks |
| タスク編集 | タスク編集モーダルを表示し、タスク情報を更新する | PUT /api/projects/{projectId}/tasks/{taskId} |
| ドラッグアンドドロップ | タスクカードを別Status列へ移動する | PATCH /api/projects/{projectId}/tasks/{taskId}/status |
| タスク選択 | タスク詳細画面へ遷移する | GET /api/projects/{projectId}/tasks/{taskId} |
| タスクステータス変更 | タスクを別Statusへ移動する | PATCH /api/projects/{projectId}/tasks/{taskId}/status |
| タスク削除 | タスクを論理削除する | DELETE /api/projects/{projectId}/tasks/{taskId} |

### タスクカード表示項目

| 項目 | 内容 |
|---|---|
| Summary | タスク概要 |
| Assignee | 担当者 |
| Priority | 優先度 |
| Due date | 期限 |

### 表示制御

| 条件 | 制御 |
|---|---|
| システム管理者 | プロジェクト編集画面へアクセス、タスク作成、編集、移動、削除が可能 |
| プロジェクト管理者 | プロジェクト編集画面へアクセス、タスク作成、編集、移動、削除が可能 |
| メンバー | タスク作成、編集、移動が可能。削除は不可 |
| 閲覧者 | タスク閲覧のみ可能 |
| 権限なし | カンバン画面へアクセス不可 |

### 空状態

対象Statusにタスクがない場合、その列にタスクが存在しない旨を表示する。

### 補足

タスク作成・編集は専用画面ではなくモーダルで行う。

初期MVPでは、ドラッグアンドドロップによるStatus変更を行う。

ドラッグアンドドロップ時は、移動先Statusをもとにタスクステータス変更APIを呼び出す。

## 18. SCR-011 招待参加画面

### 画面目的

招待URLからアクセスしたユーザーが、招待内容を確認し、対象プロジェクトへ参加する。

### 表示項目

| 項目 | 内容 |
|---|---|
| プロジェクト名 | 招待先プロジェクト名 |
| 招待先メールアドレス | 招待されたメールアドレス |
| 招待状態 | PENDING / ACCEPTED / EXPIRED / REVOKED |
| 登録フォーム | 未登録ユーザー用の表示名、パスワード入力 |
| 参加ボタン | 登録済みユーザーが招待を承認する |
| ログイン誘導 | 未ログイン時にログインを促す |

### 操作

| 操作 | 内容 | 利用API |
|---|---|---|
| 招待情報取得 | tokenから招待情報を取得する | GET /api/invitations/{token} |
| 未登録ユーザー登録・承認 | ユーザー登録後、プロジェクトへ参加する | POST /api/invitations/{token}/register |
| 登録済みユーザー承認 | ログイン済みユーザーが招待を承認する | POST /api/invitations/{token}/accept |

### 表示制御

| 条件 | 制御 |
|---|---|
| 招待状態がPENDING | 参加操作を表示する |
| 招待状態がACCEPTED | 招待が承認済みである旨を表示する |
| 招待状態がEXPIRED | 招待期限切れである旨を表示する |
| 招待状態がREVOKED | 招待が無効である旨を表示する |
| 未登録ユーザー | 登録フォームを表示する |
| 登録済みユーザー | 参加ボタンを表示する |
| ログイン中ユーザーのメールアドレスが招待先と異なる | 承認不可メッセージを表示する |

## 24. 決定事項

- プロジェクト作成画面とプロジェクト編集画面は分ける
- プロジェクト作成権限は、システム管理者のみとする
- プロジェクト作成直後は、作成したシステム管理者を対象プロジェクトのプロジェクト管理者とする
- システム管理者は、作成後に任意のユーザーをプロジェクト管理者へ変更できる
- タスク作成・編集はモーダルで行う
- カンバン画面ではドラッグアンドドロップによるStatus変更を初期MVPに含める
