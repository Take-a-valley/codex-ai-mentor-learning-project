# 要件定義書

## システム概要

本システムは、小規模開発チーム向けの開発支援アプリケーションである。

システム開発に必要なタスク管理、ドキュメント管理、進捗管理、チーム内コミュニケーションを段階的に統合することを目的とする。

初期MVPでは、複数プロジェクトに対応したカンバン方式のタスク管理機能を対象とする。

## 開発目的・背景

開発プロジェクトでは、タスク管理、ドキュメント管理、進捗管理、チャットが複数ツールに分散しやすい。

本システムでは、まずカンバン方式のタスク管理を実装し、その後Wiki、WBS、プロジェクト内チャットを段階的に追加する。

また、本プロジェクトは学習目的も含み、要件定義、設計、実装、レビュー、保守の流れを理解することを目的とする。

## 想定利用者

想定利用者は小規模開発チームである。

利用ロールは以下の4種類とする。

- システム管理者
- プロジェクト管理者
- メンバー
- 閲覧者

- メールログイン

## ロール定義

本システムでは、システム全体に関するロールと、プロジェクト内でのロールを分けて扱う。

### システム管理者

システム全体を管理するユーザー。

全ユーザー、全プロジェクトを確認できる。  
必要に応じて、ユーザー管理、プロジェクト確認、システム全体の管理操作を行う。

### プロジェクト管理者

特定プロジェクト内の管理者。

対象プロジェクトの設定、メンバー管理、ロール変更、タスク削除、コメント削除を行える。

### メンバー

対象プロジェクト内で、タスク作成、編集、ステータス移動、コメント作成、自分のコメント編集を行える。

### 閲覧者

対象プロジェクト内で、プロジェクト、タスク、コメントを閲覧できる。編集操作は行えない。

## 機能詳細

### 認証

ユーザーはメールアドレスとパスワードでログインできる。

パスワードを忘れた場合、リセット用URLをメールで受け取れる。リセットURLの有効期限は30分とする。

### プロジェクト管理

複数のプロジェクトを作成・切り替えできる。

プロジェクト作成はシステム管理者のみ可能とする。

プロジェクト作成直後は、作成したシステム管理者を対象プロジェクトのプロジェクト管理者として登録する。

システム管理者は、作成後に任意のユーザーを対象プロジェクトのプロジェクト管理者へ変更できる。

システム管理者は、システム全体の管理目的で全プロジェクトを確認できる。

プロジェクト内の管理操作は、システム管理者または対象プロジェクトのプロジェクト管理者のみ可能とする。

### メンバー招待

システム管理者及びプロジェクト管理者はメールアドレスを指定してユーザーを招待できる。

招待URLの有効期限は24時間とする。

未登録ユーザーは招待URLからユーザー登録後、対象プロジェクトに参加できる。

招待時の初期ロールはメンバー固定とする。

### 初期管理者

初期MVPでは、開発環境用の初期データとしてシステム管理者を作成する。

初期パスワードは要件定義書には記載せず、環境変数または初期セットアップ時に設定する。

システム管理者は、必要に応じて最初のプロジェクト作成やプロジェクト管理者の設定を行う。

### ロール管理

システム管理者及びプロジェクト管理者はメンバーのロールを変更できる。

システム管理者自身のロール変更は、他にシステム管理者が存在する場合のみ可能。

プロジェクト管理者自身のロール変更は、対象プロジェクト内に他のプロジェクト管理者が存在する場合のみ可能とする。

### タスク管理

タスクはカンバン方式で管理する。

タスクには以下の情報を持たせる。

- Summary
- Description
- Status
- Assignee
- Due date
- Priority
- Creator
- Created date
- Updated date

Statusは以下の4つとする。

- ToDo
- In Progress
- Review
- Done

Due dateには未来の日付のみ選択可能とし、入力は任意とする。

Priorityの選択肢は以下のとおりとする。

- Highest
- High
- Medium
- Low
- Lowest

メンバーはタスクの作成、編集、移動ができる。タスク削除はシステム管理者及びプロジェクト管理者のみ可能とする。

### コメント管理

メンバーはコメントを作成できる。

メンバーは自分のコメントのみ編集できる。

コメント削除はシステム管理者及びプロジェクト管理者のみ可能とする。

閲覧者はコメントの閲覧のみできる。

### 添付ファイル管理

タスクにファイルを添付できる。

添付ファイルはアップロード、ダウンロード、削除に対応する。

添付ファイル削除はシステム管理者及びプロジェクト管理者のみ可能とする。

初期MVPでは、添付ファイル本体をDBに保存する。

1ファイルあたりの容量上限は5MBとする。

許可するファイル種別は以下とする。

- 画像
- PDF
- テキスト
- Office系ファイル

## ファイル添付要件

- 1ファイルあたりの容量上限は5MBとする
- 添付ファイルは、ファイル名、ファイル種別、ファイルサイズ、ファイル本体データをDBで管理する

## 非機能要件

- 権限のない操作は実行できないこと
- 管理者が0人になる状態を防ぐこと
- 添付ファイルの容量と種別を制限すること
- パスワードリセットURLと招待URLには有効期限を設けること
- 将来の機能追加を考慮し、機能ごとの責務を分けること
