# フロントエンド状態管理設計書

## 1. 本書の目的

本書は、タスク管理機能の初期MVPにおけるフロントエンド状態管理方針を定義することを目的とする。

画面表示、API取得データ、入力フォーム、モーダル、ローディング、エラー、認証状態、CSRFトークンの扱いを整理する。

## 2. 対象範囲

対象に含めるものは以下とする。

- 状態管理の基本方針
- サーバー状態と画面状態の分類
- 認証状態
- CSRFトークン
- プロジェクト状態
- カンバン状態
- タスク詳細状態
- コメント状態
- 添付ファイル状態
- メンバー管理状態
- フォーム状態
- モーダル状態
- ローディング状態
- エラー状態

対象外とするものは以下とする。

- 個別コンポーネントの実装コード
- APIレスポンスの詳細定義
- CSS設計
- E2Eテスト設計

## 3. 前提方針

フロントエンドでは、状態を以下に分けて管理する。

| 分類 | 内容 | 管理方針 |
|---|---|---|
| サーバー状態 | APIから取得するデータ | TanStack Queryで管理する |
| 画面状態 | モーダル開閉、選択中項目、入力中フォーム | React標準で管理する |
| 認証状態 | ログインユーザー情報 | ログインユーザー取得APIの結果を正とする |
| 一時状態 | ローディング、エラー、送信中 | 画面または処理単位で管理する |

初期MVPでは、状態管理を過度に複雑にしない。

## 4. 採用方針

サーバー状態の管理には、TanStack Queryを使用する。

理由は以下とする。

- API取得データのキャッシュを扱いやすい
- 再取得、ローディング、エラー状態を管理しやすい
- 作成・更新・削除後の再取得を整理しやすい
- カンバン、タスク詳細、メンバー一覧などAPI依存の画面と相性がよい

画面内だけで完結する状態は、React標準の `useState` や `useReducer` を使用する。

## 5. 状態分類

| 状態 | 分類 |
|---|---|
| ログインユーザー | サーバー状態 |
| CSRFトークン | サーバー状態または共通状態 |
| プロジェクト一覧 | サーバー状態 |
| プロジェクト詳細 | サーバー状態 |
| カンバンタスク一覧 | サーバー状態 |
| タスク詳細 | サーバー状態 |
| コメント一覧 | サーバー状態 |
| 添付ファイル一覧 | サーバー状態 |
| メンバー一覧 | サーバー状態 |
| 招待情報 | サーバー状態 |
| 入力フォーム | 画面状態 |
| モーダル開閉 | 画面状態 |
| 選択中タスク | 画面状態 |
| ドラッグ中タスク | 画面状態 |
| エラー表示 | 一時状態 |
| 送信中状態 | 一時状態 |

## 6. 認証状態管理

ログイン状態はバックエンドのセッションを正とする。

フロントエンドでは、ログインユーザー情報取得APIの結果を保持する。

| 状態 | 内容 |
|---|---|
| currentUser | ログイン中ユーザー |
| isAuthenticated | ログイン済みか |
| systemRole | システムロール |
| projectRole | 選択中プロジェクトにおけるロール。プロジェクト一覧またはプロジェクト詳細の `myProjectRole` をもとに判定する |

未ログインの場合、認証必須画面ではログイン画面へ誘導する。

ログアウト時はログアウトAPIを呼び出し、保持しているユーザー情報を破棄する。

## 7. CSRFトークン管理

CSRFトークンは、状態変更APIを呼び出す前に取得する。

対象メソッドは以下とする。

- POST
- PUT
- PATCH
- DELETE

取得したCSRFトークンは、APIクライアントで状態変更リクエストに付与する。

CSRFトークンが無効または不足している場合、バックエンドは403を返却する。

## 8. プロジェクト状態管理

プロジェクト一覧画面では、プロジェクト一覧をサーバー状態として管理する。

| 状態 | 内容 |
|---|---|
| projects | プロジェクト一覧 |
| selectedProjectId | 選択中プロジェクトID |
| projectDetail | プロジェクト詳細 |
| projectMembers | プロジェクトメンバー一覧 |

プロジェクト作成・編集・削除後は、プロジェクト一覧または詳細を再取得する。

## 9. カンバン状態管理

カンバン画面では、Statusごとにタスク一覧を管理する。

| 状態 | 内容 |
|---|---|
| board | Statusごとのタスク一覧 |
| activeTaskId | 選択中またはドラッグ中のタスクID |
| selectedStatus | 選択中Status |
| isDragging | ドラッグ中かどうか |

タスクをドラッグアンドドロップした場合、移動先StatusをAPIへ送信する。

初期MVPでは、同一Status内の並び替えは対象外とする。

Status変更後は、カンバン一覧を再取得する。

## 10. タスク作成・編集フォーム状態

タスク作成・編集モーダルでは、入力中のフォーム値を画面状態として管理する。

| 状態 | 内容 |
|---|---|
| summary | タスク概要 |
| description | タスク説明 |
| status | Status |
| priority | Priority |
| assigneeId | 担当者ID |
| dueDate | 期限 |

送信成功後はモーダルを閉じ、カンバンまたはタスク詳細を再取得する。

送信失敗時は、入力内容を保持したままエラーを表示する。

## 11. タスク詳細状態管理

タスク詳細画面では、タスク詳細、コメント、添付ファイルをサーバー状態として管理する。

| 状態 | 内容 |
|---|---|
| taskDetail | タスク詳細 |
| comments | コメント一覧 |
| attachments | 添付ファイル一覧 |

コメント作成、コメント編集、コメント削除、添付追加、添付削除後は、タスク詳細または該当一覧を再取得する。

## 12. コメント状態管理

コメント入力欄の内容は画面状態として管理する。

| 状態 | 内容 |
|---|---|
| commentBody | 入力中コメント |
| editingCommentId | 編集中コメントID |
| editingCommentBody | 編集中コメント本文 |

コメント作成・編集成功後は、入力状態を初期化する。

コメント削除成功後は、コメント一覧を再取得する。

## 13. 添付ファイル状態管理

添付ファイルアップロードでは、選択中ファイルと送信状態を画面状態として管理する。

| 状態 | 内容 |
|---|---|
| selectedFile | 選択中ファイル |
| uploadError | アップロードエラー |
| isUploading | アップロード中かどうか |

フロントエンドでは、ファイルサイズとファイル種別を事前確認する。

最終判定はバックエンドで行う。

## 14. メンバー管理状態

メンバー管理画面では、メンバー一覧をサーバー状態として管理する。

| 状態 | 内容 |
|---|---|
| members | プロジェクトメンバー一覧 |
| changingRoleMemberId | ロール変更中メンバーID |
| deletingMemberId | 削除中メンバーID |

ロール変更やメンバー削除後は、メンバー一覧を再取得する。

PROJECT_ADMINが0人になる操作はバックエンドで禁止する。

## 15. 招待状態管理

メンバー招待画面では、招待フォームを画面状態として管理する。

| 状態 | 内容 |
|---|---|
| email | 招待先メールアドレス |
| inviteError | 招待エラー |
| isInviting | 招待送信中かどうか |

招待送信成功後は、完了メッセージを表示する。

招待参加画面では、招待情報をサーバー状態として取得する。

## 16. モーダル状態管理

モーダルの開閉状態は画面状態として管理する。

| モーダル | 状態 |
|---|---|
| タスク作成モーダル | isTaskCreateOpen |
| タスク編集モーダル | editingTaskId |
| 削除確認ダイアログ | deletingTarget |
| コメント編集フォーム | editingCommentId |

モーダルを閉じる場合、入力中の値を破棄するか確認する。

## 17. ローディング状態管理

ローディング状態は、処理単位で管理する。

| 状態 | 表示方針 |
|---|---|
| 初期取得中 | 画面全体または主要領域に表示する |
| 一覧再取得中 | 対象領域に表示する |
| 送信中 | 送信ボタンを無効化する |
| アップロード中 | 添付ファイル領域に表示する |

二重送信を防ぐため、送信中の操作ボタンは無効化する。

## 18. エラー状態管理

APIエラーは、画面または部品単位で保持する。

| エラー | 表示方針 |
|---|---|
| 入力エラー | 対象フォーム付近に表示する |
| 認証エラー | ログイン画面へ誘導する |
| 認可エラー | 権限がない旨を表示する |
| データなし | 対象データが存在しない旨を表示する |
| 業務制約エラー | APIのメッセージを表示する |
| システムエラー | 汎用エラーメッセージを表示する |

エラー表示後も、ユーザーが次の操作を理解できる状態にする。

## 19. 再取得方針

データ更新後は、関連するサーバー状態を再取得する。

| 操作 | 再取得対象 |
|---|---|
| プロジェクト作成 | プロジェクト一覧 |
| プロジェクト更新 | プロジェクト一覧、プロジェクト詳細 |
| プロジェクト削除 | プロジェクト一覧 |
| タスク作成 | カンバン一覧 |
| タスク更新 | カンバン一覧、タスク詳細 |
| タスクStatus変更 | カンバン一覧 |
| タスク削除 | カンバン一覧 |
| コメント作成・編集・削除 | タスク詳細またはコメント一覧 |
| 添付追加・削除 | タスク詳細または添付ファイル一覧 |
| メンバーロール変更 | メンバー一覧 |
| メンバー削除 | メンバー一覧 |

## 20. キャッシュ方針

サーバー状態は、画面表示に必要な範囲でキャッシュする。

ただし、権限やタスク状態は変更される可能性があるため、更新操作後は再取得を行う。

古い権限情報で操作ボタンが表示されないように、ロール変更後はメンバー一覧やログインユーザー情報の再取得を検討する。

## 21. 権限表示状態

フロントエンドでは、ログインユーザー情報とプロジェクトロールをもとに表示制御を行う。

| 操作 | 表示条件 |
|---|---|
| プロジェクト作成 | SYSTEM_ADMIN |
| プロジェクト編集 | SYSTEM_ADMIN または PROJECT_ADMIN |
| メンバー管理 | SYSTEM_ADMIN または PROJECT_ADMIN |
| タスク作成 | SYSTEM_ADMIN、PROJECT_ADMIN、MEMBER |
| タスク編集 | SYSTEM_ADMIN、PROJECT_ADMIN、MEMBER |
| タスク削除 | SYSTEM_ADMIN、PROJECT_ADMIN |
| コメント作成 | SYSTEM_ADMIN、PROJECT_ADMIN、MEMBER |
| コメント編集 | 自分のコメント |
| コメント削除 | SYSTEM_ADMIN、PROJECT_ADMIN |
| 添付追加 | SYSTEM_ADMIN、PROJECT_ADMIN、MEMBER |
| 添付削除 | SYSTEM_ADMIN、PROJECT_ADMIN |

最終的な認可判定はバックエンドで行う。

## 22. 決定事項

- サーバー状態と画面状態を分けて管理する
- サーバー状態はTanStack Queryで管理する
- 画面状態はReact標準の状態管理を使用する
- ログイン状態はバックエンドのセッションを正とする
- CSRFトークンはAPIクライアントで扱う
- タスク作成・編集モーダルの入力値は画面状態として管理する
- 更新操作後は関連データを再取得する
- 認可の最終判定はバックエンドで行う
